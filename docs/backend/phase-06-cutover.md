# Phase 06 - Cutover

## 이 단계 목표
- 런타임 인증 의존을 레거시 스택에서 Spring으로 완전히 전환합니다.

## 초보자 핵심 개념
- "컷오버"는 기능 구현보다 의존성 정리가 핵심입니다.
- 코드/설정/문서 3가지를 함께 바꿔야 실제 전환이 완료됩니다.

## 왜 이 설계를 선택했는가
- 인증 체크를 프론트 프록시에서 Spring `/auth/me` 호출로 통일했습니다.
- 레거시 인증 코드를 런타임에서 완전히 제거해 인증 소스를 단일화했습니다.

## 코드 구조 지도(파일 역할)
- `src/proxy.ts`: 보호 라우트 인증 확인
- `.env.example`: 프론트 백엔드 API 기준으로 변경

## 요청/응답 예시(JSON)
보호 라우트 접근시 내부 체크:
```json
{
  "check": "GET /auth/me",
  "authenticated": true,
  "action": "allow"
}
```

## 실패 케이스와 디버깅 포인트
- 보호 라우트 무한 리다이렉트: `/auth/me` 401 원인 확인(쿠키/도메인/CORS)
- 빌드 오류: 제거된 라이브러리 import 잔존 여부 확인
